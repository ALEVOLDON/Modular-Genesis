## Website Purpose

The site is intended to do more than publish text. It should make the system legible.

That means it should present:

- roadmap
- lessons
- patch library
- visual experiments
- system diagrams

## Visual Direction

The design language points toward:

- modular interface inspiration
- digital lab or studio feeling
- dark background with bright signal accents
- diagram-first presentation

## Key Pages

The main public areas are:

- landing page
- roadmap
- lessons
- patch library
- visual experiments
- resources
