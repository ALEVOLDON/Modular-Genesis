# Course And Repository Vision

This document consolidates the source notes that describe `Modular Genesis` as a public learning project.

## Project Intent

The repository is not only meant to hold patches. It is meant to grow into:

- an open course
- a public reference library
- a structured patch archive
- a site for modular and audiovisual education

## Educational Direction

The note set proposes a staged curriculum rather than a pile of isolated tutorials.

The recurring levels are:

1. Foundations
2. Sequencing
3. Generative Systems
4. Hybrid Setup
5. Audiovisual Performance

This progression is useful because it maps cleanly to both:

- documentation folders
- public site sections

## Repository As Product

The notes repeatedly point toward a documentation-first repository that could contain:

- lessons
- diagrams
- downloadable `VCV Rack` patches
- `Ableton Live` templates
- visual experiments
- media examples

That means the repository itself should be understandable even before a full site exists.

## Lesson Format

A stable lesson format appears across the planning notes:

- theory
- patch diagram
- real patch file
- short demo
- task or exercise

This is the correct structure for turning the current material into an actual learning system.

## Public Value

The notes position the project as something larger than a personal notebook:

- a reusable education resource
- a bridge between modular synthesis and audiovisual practice
- a public technical reference for artists and learners

That is why naming, structure, and consistency matter at this stage.
