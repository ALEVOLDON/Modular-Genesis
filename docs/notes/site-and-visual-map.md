# Site And Visual Map

This document consolidates the planning notes related to the project website, visual identity, and interaction model.

## Website Purpose

The future site is intended to do more than publish text. It should make the system legible.

That means it should present:

- roadmap
- lessons
- patch library
- visual experiments
- system diagrams

## Visual Direction

The source notes repeatedly point toward a technical but atmospheric design language:

- modular interface inspiration
- digital lab or studio feeling
- dark background with bright signal accents
- diagram-first presentation

The design should feel intentional and system-oriented, not generic educational UI.

## Key Pages

The notes repeatedly suggest the following public areas:

- landing page
- roadmap
- lessons
- patch library
- visual experiments
- resources

That structure is now reflected in the initialized website scaffold.

## Visual Map Concept

One of the stronger ideas in the notes is a central architecture map:

```text
Controller
  -> Ableton Live
  -> VCV Rack
  -> Generative System
  -> Audio Engine
  -> Visual Engine
  -> Performance Output
```

This map can function as:

- homepage illustration
- navigation aid
- high-level learning diagram

## Browser Interaction Idea

The notes also suggest an eventual interactive layer:

- clickable modules
- audio examples
- patch references
- WebAudio or WebGL prototypes

That is a later phase, but it should inform how pages and content are structured now.
